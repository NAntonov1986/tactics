# Фэнтези-тактика — гайд для Claude

Проект — браузерная пошаговая тактика на ванильном JS (без сборщика). Точка входа — [index.html](index.html), которая подключает модули из [src/](src/). Жанр: X-COM-like с фэнтезийным сеттингом, мета-слой лагеря + тактические миссии на сетке 20×20.

## Ключевые документы

- **[DESIGN.md](DESIGN.md)** — рабочий дизайн-документ. Видение, дизайн-философия, roadmap, архитектурные инварианты, активные планы, edge-case'ы, политика бекапов. **Сюда смотреть, когда вопрос «куда движемся / зачем так сделано / что планируется».**
- **[CODEX.md](CODEX.md)** — справочник по текущему состоянию игры: что уже реализовано, какие сущности есть, как они считаются. Обновляется при каждом изменении правил. **Сюда смотреть, когда вопрос «как сейчас работает / какие формулы / что в коде».**
- **[archive/REFACTOR_LOG.md](archive/REFACTOR_LOG.md)** — история рефакторинга (R0–R19).

Разделение строгое: планы и философия — в DESIGN.md, факты о текущем коде — в CODEX.md.

## Структура проекта

- [index.html](index.html) — точка входа, грузит [src/main.js](src/main.js).
- [src/main.js](src/main.js) — bootstrap.
- [src/core/](src/core/) — игровая логика: бой, ИИ, движение, эффекты, повышение уровней, лут, ходы, save/load, state, расчёт статов, скиллы.
- [src/data/](src/data/) — статические данные: классы, оружие, аффиксы, расходники, экипировка, скиллы, статы, типы юнитов, типы урона, экономика.
- [src/render/](src/render/) — отрисовка: поле, юниты, панели, модалки, инвентарь, лог, оверлеи, тултипы.
- [src/ui/](src/ui/) — пользовательский ввод: хоткеи, инпут, тултипы, зум/пан, dev-tools, debug-log, скорость анимаций.
- [styles/](styles/) — CSS по экранам/компонентам (base, board, camp, panel, inventory, level-up, log, tooltip, fantasy-тема и др.).
- [assets/sprites/](assets/sprites/) — спрайты.
- [tools/](tools/) — вспомогательные скрипты (например, `gen-skills.js`).
- [archive/](archive/) — архив завершённых сессий и логов рефакторинга.
- [backup/](backup/) — бекапы.

## Правила работы

**Пользователь учится JavaScript.** К каждой нетривиальной правке коротко поясняй:
- что делает блок (1-2 предложения);
- какие идиомы/конструкции JS используются (деструктуризация, spread, замыкания, `?.`/`??`, методы массивов, async/await, классы, прототипы и т.п.) — называй их по имени, чтобы пользователь учил терминологию.

Тривиальные правки (переименование, очевидный фикс опечатки, перенос строки) — без пояснений. Если приём уже разбирался ранее в той же сессии — повторять не нужно.
